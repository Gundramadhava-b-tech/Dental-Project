# OSA Diagnostic – Baseline / Load Testing

## Overview

This directory contains everything needed to run a **Baseline Load Test** against the OSA Diagnostic API and export the results to a styled **Excel workbook**.

| Parameter        | Value                          |
|------------------|-------------------------------|
| Virtual Users    | **100 concurrent users**      |
| Duration         | **1 minute** (60 seconds)     |
| Test type        | Baseline / Load test           |
| Tool             | [k6](https://k6.io)           |

---

## Prerequisites

### 1. k6 (load testing tool)

```powershell
# Option A – winget (Windows built-in)
winget install k6

# Option B – Chocolatey
choco install k6

# Option C – Download installer
# https://k6.io/docs/get-started/installation/
```

### 2. Node.js

Download from https://nodejs.org (LTS version recommended).

### 3. Install npm dependencies (exceljs)

```powershell
cd load-tests
npm install
```

---

## Quick Start  (Recommended)

Run the **all-in-one PowerShell script**:

```powershell
# From the project root
.\load-tests\run-load-test.ps1

# Against a remote server
.\load-tests\run-load-test.ps1 -BaseUrl "https://your-api.example.com/api"
```

This script:
1. Checks that `k6` and `node` are installed
2. Installs `exceljs` if not already present
3. Runs the k6 load test and saves raw results to `results.json`
4. Generates `load-test-report.xlsx` with 5 formatted sheets

---

## Manual Steps

### Step 1 – Start the backend server

```powershell
# From project root
pnpm dev
```

Or make sure your backend is accessible at `http://localhost:5000`.

### Step 2 – Run the k6 load test

```powershell
# From project root  (saves JSON to load-tests/results.json)
k6 run --out json=load-tests/results.json load-tests/baseline.js

# Against a remote URL
k6 run --out json=load-tests/results.json ^
  --env BASE_URL=https://your-api.com/api ^
  load-tests/baseline.js
```

### Step 3 – Generate the Excel report

```powershell
cd load-tests
node generate-excel.js
# output: load-tests/load-test-report.xlsx
```

## What the Test Does

Each virtual user runs the following loop for 60 seconds:

```
GET /api/healthz           → check API is alive
  sleep 0.3s
GET /api/patients          → list all patients (uses x-physician-id header)
  sleep 0.3s
GET /api/patients/:id      → fetch first patient detail
  sleep 0.3s
GET /api/scans             → list all scans
  sleep 0.3s
GET /api/analyses          → list all analyses
  sleep 0.3s
```

## Performance Thresholds

| Threshold                        | Limit         | Meaning                                   |
|----------------------------------|---------------|-------------------------------------------|
| `http_req_duration` p(95)        | < 2000 ms     | 95% of requests under 2s                 |
| `http_req_duration` p(99)        | < 3000 ms     | 99% of requests under 3s                 |
| `health_duration` p(95)          | < 500 ms      | Health endpoint very fast                 |
| `patients/scans/analyses` p(95)  | < 2000 ms     | All data endpoints under 2s              |
| `error_rate`                     | < 5%          | Less than 5% errors                      |
| `checks` pass rate               | > 95%         | At least 95% of assertions pass          |

## Reading the Results

### Requests per Second (RPS)
```
http_reqs: 7200/min  →  120 req/sec
```
Your API is handling ~120 requests every second.

### Response Time Example
```
avg: 250ms   min: 50ms   max: 1500ms
p(95): 480ms   p(99): 950ms
```
- Fastest response: 50 ms  
- Average: 250 ms  
- Slowest: 1.5 s  
- 95% of requests: under 480 ms

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `k6: command not found` | Install k6 via winget or Chocolatey |
| `Missing x-physician-id header` | The test sends `PHYSICIAN_ID` env var; set it if needed |
| `Cannot connect to localhost:5000` | Start the backend server first |
| `results.json not found` | Run k6 before running the Excel generator |
| Excel file won't open | Install Microsoft Excel or LibreOffice |
